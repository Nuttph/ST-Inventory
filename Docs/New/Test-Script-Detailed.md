# Test Script: ระบบ ST-Inventory (Detailed Version)

เอกสารนี้รวบรวม Test Script แยกตามฟังก์ชันหลักของระบบ โดยอ้างอิงขั้นตอนและฟิลด์ข้อมูล (Input Fields) จากซอร์สโค้ดและ Business Logic ของแอปพลิเคชัน

---

## 1. Test Script | Login (เข้าสู่ระบบ)
**ฟังก์ชัน:** ตรวจสอบสิทธิ์การเข้าใช้งาน
**API Reference:** `POST /api/auth/login`

| Step | คำอธิบายขั้นตอน | เงื่อนไข / Form Validation |
| :--- | :--- | :--- |
| **1** | เปิดหน้าเว็บไซต์และไปที่เมนู **"เข้าสู่ระบบ" (Login)** | หน้าจอต้องแสดงฟิลด์ Email และ Password |
| **2** | กรอกข้อมูลในช่อง **Email** | - ต้องไม่เป็นค่าว่าง<br>- ต้องเป็นรูปแบบ Email เท่านั้น (เช่น name@domain.com) |
| **3** | กรอกข้อมูลในช่อง **Password** | - ต้องไม่เป็นค่าว่าง<br>- ระบบจะซ่อนตัวอักษรขณะพิมพ์ (Masking) |
| **4** | คลิกปุ่ม **"เข้าสู่ระบบ" (Login)** | - ระบบตรวจสอบข้อมูลในฐานข้อมูล (User collection)<br>- หากข้อมูลถูกต้อง จะได้รับ Auth Token และ Redirect ไปหน้า Shop |

---

## 2. Test Script | Register (สมัครสมาชิก)
**ฟังก์ชัน:** สร้างบัญชีผู้ใช้ใหม่ในระบบ
**API Reference:** `POST /api/auth/register`

| Step | คำอธิบายขั้นตอน | เงื่อนไข / Form Validation |
| :--- | :--- | :--- |
| **1** | คลิกปุ่ม **"สมัครสมาชิก" (Register)** | แสดงฟอร์มการกรอกข้อมูล Name, Email, Password, Confirm Password |
| **2** | กรอกข้อมูลชื่อ-นามสกุล ในช่อง **Name** | - ต้องไม่เป็นค่าว่าง<br>- รองรับทั้งภาษาไทยและอังกฤษ |
| **3** | กรอกข้อมูลในช่อง **Email** | - ต้องเป็นรูปแบบ Email ที่ถูกต้อง<br>- ระบบตรวจสอบว่า Email นี้ต้องยังไม่เคยมีในระบบ (Unique Constraint) |
| **4** | กรอกข้อมูลในช่อง **Password** | - ต้องมีความยาวตามเงื่อนไข (เช่น 6 ตัวอักษรขึ้นไป)<br>- ต้องไม่เป็นค่าว่าง |
| **5** | กรอกข้อมูลในช่อง **Confirm Password** | - ข้อมูลต้องตรงกับช่อง Password ทุกประการ |
| **6** | เลือกบทบาทผู้ใช้งาน (ถ้ามีในหน้าจอสมัครสำหรับ Staff) | - ค่าเริ่มต้น (Default) คือ 'member' |
| **7** | คลิกปุ่ม **"สมัครสมาชิก" (Register)** | - ตรวจสอบการบันทึกลง MongoDB (User Schema)<br>- เมื่อสำเร็จต้องย้ายไปหน้า Login |

## 3. Test Script | Create Product (เพิ่มสินค้าใหม่)
**ฟังก์ชัน:** เพิ่มรายการสินค้าเข้าสู่คลังสินค้า (สำหรับ Admin/Staff เท่านั้น)
**API Reference:** `POST /api/products`

| Step | คำอธิบายขั้นตอน | เงื่อนไข / Form Validation |
| :--- | :--- | :--- |
| **1** | เข้าสู่ระบบด้วยสิทธิ์ Staff/Manager และไปหน้า **Inventory** | ต้องมีปุ่มสำหรับ "Add New Product" |
| **2** | กรอกชื่อสินค้าในช่อง **Name** | - ต้องมีค่า (Required)<br>- ความยาวไม่เกินที่กำหนด (เช่น 100 ตัวอักษร) |
| **3** | กรอกราคาสินค้าในช่อง **Price** | - ต้องเป็นตัวเลขเท่านั้น (Number)<br>- ค่าต้องมากกว่า 0 |
| **4** | กรอกจำนวนสินค้าในช่อง **Stock** | - ต้องเป็นตัวเลขจำนวนเต็ม (Integer)<br>- ค่าเริ่มต้นคือ 0 หรือค่ามมากกว่า |
| **5** | เลือกหมวดหมู่สินค้าในช่อง **Category** | - ต้องเลือกจากรายการที่กำหนด (Required) |
| **6** | (Optional) ใส่ URL รูปภาพในช่อง **Image** | - หากไม่กรอก ระบบจะใช้รูปภาพเริ่มต้น (Default Image URL) |
| **7** | คลิกปุ่ม **"บันทึก" (Save/Create)** | - ระบบบันทึกลง Product Collection<br>- รายชื่อสินค้าใหม่ต้องปรากฏในตาราง Inventory ทันที |

## 4. Test Script | Create Order & Payment (สั่งซื้อและชำระเงิน)
**ฟังก์ชัน:** การทำงานต่อเนื่องจากการเลือกสินค้าจนถึงการตัดสต็อก
**API Reference:** `POST /api/orders` & `POST /api/payments`

| Step | คำอธิบายขั้นตอน | เงื่อนไข / Form Validation |
| :--- | :--- | :--- |
| **1** | เลือกสินค้าลงใน **Cart** และกด **Checkout** | ระบบต้องดึงราคาสินค้าปัจจุบันมาคำนวณ |
| **2** | ตรวจสอบรายการในตะกร้า (Cart Review) | - จำนวนสินค้า (Quantity) ต้องไม่เกินยอดคงเหลือใน Stock |
| **3** | กรอก/เลือกที่อยู่จัดส่ง **Address** | - ต้องไม่เป็นค่าว่าง<br>- ต้องมีเบอร์โทรศัพท์ติดต่อ |
| **4** | เลือกช่องทางการชำระเงิน **Payment Method** | - ตัวเลือก: QR Code, Card, COD (ตาม Order Model) |
| **5** | กดยืนยันคำสั่งซื้อ (Create Order) | - ระบบสถานะ Order เป็น 'pending' |
| **6** | จำลองการชำระเงิน (Submit Payment) | - สถานะ Order เปลี่ยนจาก 'pending' เป็น 'paid'<br>- **เงื่อนไขสำคัญ:** ระบบต้องทำการลดจำนวน Stock สินค้าอัตโนมัติ |

---

## 5. Test Script | Update Order Status (จัดการสถานะคำสั่งซื้อ)
**ฟังก์ชัน:** ขั้นตอนการเตรียมและจัดส่งสินค้า (สำหรับ Staff)
**API Reference:** `PUT /api/orders/:id`

| Step | คำอธิบายขั้นตอน | เงื่อนไข / Form Validation |
| :--- | :--- | :--- |
| **1** | ไปหน้า **Order Management** | แสดงรายการคำสั่งซื้อเรียงตามวันที่ (Newest first) |
| **2** | เลือก Order ที่มีสถานะ 'paid' | Staff ตรวจสอบสินค้าที่ต้องบรรจุ |
| **3** | เปลี่ยนสถานะเป็น **'packing'** | ระบบบันทึกเวลาการเริ่มบรรจุสินค้า |
| **4** | กรอกเลขพัสดุ (Tracking Number) และเปลี่ยนประเภท | - ฟิลด์ Tracking ต้องไม่เป็นค่าว่างก่อนปรับเป็น 'shipped' |
| **5** | กดยืนยันการจัดส่ง **'shipped'** | - ระบบปิดการแก้ไขชื่อลูกค้า/ที่อยู่ |
| **6** | คลิกดูประวัติคำสั่งซื้อ (Orders History) | ลูกค้าต้องเห็นสถานะเป็น 'shipped' ทันที |

---

### สรุปความต้องการด้าน Validation ร่วมกัน (Global Constraints)
1. **Required Fields:** ทุกฟิลด์ที่มีเครื่องหมายดอกจัน (*) หรือ Property `required: true` ใน Model ต้องถูกตรวจสอบดักหน้าเว็บ
2. **Data Types:** ตัวราคาและจำนวนต้องป้องกันการกรอกอักษร (Input Type="number")
3. **Roles Check:** ฟังก์ชันจัดการสินค้าและคำสั่งซื้อ (Step 3, 5) จะทำได้เฉพาะผู้มี Role 'admin', 'manager', หรือ 'staff' เท่านั้น (ตรวจสอบผ่าน Middleware)
